# Ata da Reunião – Project Bart

## 1. Preparação do Ambiente de Desenvolvimento

* Cada membro deve preparar uma máquina com Linux (VM com Ubuntu ou WSL).
* Este será o ambiente oficial de desenvolvimento do projeto.

## 2. Pesquisa sobre ROS 2

* Todos os membros devem pesquisar e aprofundar conhecimentos sobre ROS 2.

## 3. Arquitetura de Software do Bart

* Iniciar a idealização da arquitetura de software tendo em conta a framework do ROS 2.
* Identificar:

  * Principais módulos (sensores, locomoção, navegação, telemetria, CLI, etc.)
  * Tipos de nós necessários
  * Fluxo de dados entre módulos
  * Dependências internas e externas

## 4. Instalação e Experimentação com ROS 2 (se possível)

* Instalar ROS 2 no ambiente preparado.
* Realizar testes simples:

  * Criar e executar nodes básicos
  * Testar "talker" e "listener"
  * Explorar comandos: `ros2 topic`, `ros2 node`, `ros2 run`, `ros2 launch`
